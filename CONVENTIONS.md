# Conventions

Patterns and standards for the Locale codebase. Updated after each implementation session.

---

## Directory Structure

```
src/
  app/          ŌåÆ Next.js App Router pages, layouts, API routes
  components/   ŌåÆ Reusable React components
  lib/          ŌåÆ Business logic, utilities, database, API clients
  types/        ŌåÆ Shared TypeScript type definitions
drizzle/        ŌåÆ Migration files (generated by drizzle-kit, do not edit manually)
docs/           ŌåÆ Product docs, build strategy, implementation plan, reviews
```

## Path Aliases

- `@/*` ŌåÆ `./src/*` (configured in `tsconfig.json`)

## Design System

- **Token location:** All design tokens live in `src/app/globals.css` inside the `@theme` block (Tailwind v4 pattern)
- **Color palette:** Warm neutrals (`cream`, `warm-50`ŌĆō`warm-950`), ink text colors, editorial green accent (`accent`, `accent-light`, `accent-muted`, `accent-subtle`), 6 muted data visualization colors
- **Fonts:** Serif headings (Playfair Display via `--font-serif`) + sans-serif body (Inter via `--font-sans`), loaded via `next/font/google` in `layout.tsx`
- **Typography:** Responsive heading sizes using `clamp()` for mobile compatibility
- **Principle:** "Magazine, not dashboard" ŌĆö editorial, warm, generous whitespace

## Database

- **ORM:** Drizzle ORM with `@neondatabase/serverless` (HTTP driver)
- **Column naming:** snake_case in PostgreSQL, camelCase in TypeScript (Drizzle maps via column name string argument)
- **Timestamps:** All tables include `created_at`. Tables with mutable rows include `updated_at` with `.$onUpdate(() => new Date())`
- **Connection:** Lazy-initialized via `getDb()` from `src/lib/db/index.ts` ŌĆö never imported at module top level to avoid build-time errors
- **Migrations:** Generated with `npm run db:generate`, applied with `npm run db:migrate`. Use `npm run db:push` for rapid local iteration

## Database (continued)

- **Foreign keys:** Use `{ onDelete: "cascade" }` when child rows should be cleaned up with their parent
- **Indexes:** Always add explicit indexes on foreign key columns (PostgreSQL does not create them automatically)
- **Migration files:** Auto-generated names (e.g., `0000_cute_scrambler.sql`) ŌĆö never rename or edit manually. Regenerating migrations produces new names.

## Linting

- **ESLint:** Flat config format at `eslint.config.mjs` (ESLint 9+)
- **Extends:** `eslint-config-next` with `core-web-vitals` + `typescript` presets

## TypeScript

- Strict mode enabled
- Prefer explicit types over `any`
- Use path alias `@/` for all imports within `src/`

## Documentation

- Architectural deviations from BUILD-STRATEGY.md are recorded in `docs/DECISIONS.md`
- Code reviews are stored in `docs/reviews/` with resolution status tracked in the file header
